# 2026-04-08 Bug修复

## DOM渲染完成前绑定事件的问题

### 问题描述：
创建角色页面的"创建角色"按钮点击无反应，导航按钮也无法切换页面。

### 问题原因：
代码在`innerHTML`设置后立即调用事件绑定函数，但DOM可能还未完全渲染完成，导致`document.getElementById()`和`document.querySelectorAll()`找不到元素。

### 修复方案：
使用`requestAnimationFrame`确保DOM渲染完成后再绑定事件。

### 修改内容：
1. `_showCharacterCreate()`中的`_bindCharacterCreateEvents()`调用：
   - 修复前：`this._bindCharacterCreateEvents();`
   - 修复后：`requestAnimationFrame(() => { this._bindCharacterCreateEvents(); });`

2. `_showMainPage()`中的`_bindNavEvents()`调用：
   - 修复前：`this._bindNavEvents();`
   - 修复后：`requestAnimationFrame(() => { this._bindNavEvents(); });`

3. 添加错误日志：
   - 在`_bindCharacterCreateEvents()`中添加元素检查，输出错误日志便于调试

### 修改的文件：
- `src/scripts/main.js`：第223行、第392行附近

### 测试结果：
- ✅ 创建角色功能正常
- ✅ 职业选择正常
- ✅ 导航切换正常
