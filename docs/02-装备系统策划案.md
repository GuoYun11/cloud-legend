# 装备系统策划案

## 1. 系统概述

装备系统管理玩家的装备穿戴、背包、套装激活、装备强化、幸运培养与材料消耗等核心玩法。装备是角色属性的主要来源，当前版本除套装、四格配饰外，已扩展出左右手镯、武器祝福油、项链通用幸运升级与材料掉落链路。

### 1.1 当前版本迭代同步

- 装备槽位已扩展为 `左戒指/右戒指/左手镯/右手镯`，并支持自动分配左右槽位。
- 武器幸运采用传奇式设计：`总幸运=7` 时普通攻击稳定打出攻击上限。武器幸运上限为7。
- 项链不再拆分为独立“幸运项链系列”，而是所有项链都可消耗金币与材料升级幸运，最高 `+5`。
- 新增材料 `幸运符石`、`祝福宝石`、`超级祝福油`，由普通怪/BOSS分层掉落。
- 装备页需要同时展示装备幸运、材料库存、祝福油入口与项链幸运升级入口。

## 2. 数据结构

### 2.1 装备数据

```typescript
// 装备实例（玩家背包中的具体装备）
interface EquipmentInstance {
  instanceId: string;          // 唯一实例ID（UUID）
  configId: string;            // 装备配置ID（引用配置表）
  quality: QualityType;        // 品质
  level: number;               // 装备等级需求
  slot: EquipSlot;             // 装备部位
  setId?: string;              // 套装ID（如属于套装）
  
  stats: EquipStats;           // 装备属性（主属性）
  extraStats: ExtraStat[];     // 附加词条（0-5条，按品质）
  
  enhanceLevel: number;        // 强化等级 (0-10)
  lucky?: number;              // 幸运值（武器0-9，项链0-5，其它部位默认为0）
  curse?: number;              // 诅咒值（预留）
  durability?: number;         // 耐久度（可选）
  
  isIdentified: boolean;       // 是否已鉴定
  isLocked: boolean;           // 是否锁定（防误删）
  obtainedAt: number;          // 获得时间
  source: DropSource;          // 来源
}

// 品质类型
type QualityType = 'normal' | 'uncommon' | 'rare' | 'epic' | 'legendary' | 'mythic';

// 装备部位
type EquipSlot = 
  | 'weapon'     // 武器
  | 'armor'      // 衣服
  | 'helmet'     // 头盔
  | 'necklace'   // 项链
  | 'leftRing'   // 左戒指
  | 'rightRing'  // 右戒指
  | 'leftBracelet'   // 左手镯
  | 'rightBracelet'  // 右手镯
  | 'belt'       // 腰带
  | 'shoes'      // 鞋子
  | 'medal'      // 勋章（四格）
  | 'hat'        // 斗笠（四格）
  | 'drum'       // 军鼓（四格）
  | 'talisman';  // 符咒（四格）

// 装备属性
interface EquipStats {
  attack?: number;
  magicAttack?: number;
  defense?: number;
  magicDefense?: number;
  maxHp?: number;
  maxMp?: number;
  critRate?: number;
  critDamage?: number;
  dodgeRate?: number;
  attackSpeed?: number;
  hpRegen?: number;
  expBonus?: number;
  goldBonus?: number;
  magicFind?: number;
  damageReduction?: number;
}

// 附加词条
interface ExtraStat {
  type: keyof EquipStats;
  value: number;
}

// 来源
type DropSource = 'monster' | 'boss' | 'quest' | 'shop' | 'craft';
```

### 2.2 穿戴槽数据

```typescript
// 玩家当前穿戴
interface PlayerEquipment {
  playerId: string;
  slots: {
    [K in EquipSlot]?: string;   // 部位 -> instanceId
  };
  // 套装激活状态（计算缓存，不存储，运行时计算）
  activeSets?: ActiveSetInfo[];
}

interface ActiveSetInfo {
  setId: string;
  setName: string;
  equippedCount: number;
  totalPieces: number;
  activeBonuses: string[];       // 当前激活的加成描述
}
```

### 2.3 背包数据

```typescript
interface Inventory {
  playerId: string;
  capacity: number;              // 背包容量（默认60格）
  items: EquipmentInstance[];    // 装备列表
}
```

### 2.4 材料背包数据

```typescript
type MaterialId = 'lucky_stone' | 'blessing_gem' | 'super_oil_item';

interface MaterialBag {
  playerId: string;
  items: Partial<Record<MaterialId, number>>;
}

interface LuckyTotalInfo {
  lucky: number;          // 武器幸运 + 项链幸运
  weaponLucky: number;
  necklaceLucky: number;
  curse: number;
  net: number;            // 幸运-诅咒后的实际值
}
```

#### 幸运规则说明

- 武器幸运只接受 `祝福油/超级祝福油` 提升，不通过材料面板直接升级。
- 项链幸运为通用成长线，所有项链均可升级，不再制作单独的幸运项链系列。
- 项链幸运 `+4` 额外提供 `暴击伤害+15%`，`+5` 额外提供 `暴击率+5%`。
- 角色战斗使用的总幸运 = `武器幸运 + 项链幸运 - 诅咒`。

- 例如：Lv10 的玩家可以穿戴需求等级为 Lv13 的装备
- 穿戴检查公式：`playerLevel >= requiredLevel - 3`
- 背包中的升级指示器（绿色▲）同样遵循此容差规则

## 5. 掉落逻辑

```typescript
class DropSystem {
  /** 执行一次掉落（怪物死亡时调用） */
  static rollDrop(mapId: string, monsterId: string, playerMagicFind: number): DropResult {
    const dropTable = DROP_TABLES[mapId]?.[monsterId];
    if (!dropTable) return { items: [], materials: [], gold: 0 };
    
    const drops: EquipmentInstance[] = [];
    const materials: MaterialDrop[] = [];
    
    // 1. 先结算材料掉落（独立于装备）
    if (dropTable.materialDrops) {
      for (const mat of dropTable.materialDrops) {
        if (Math.random() < mat.chance * (1 + playerMagicFind * 0.5)) {
          materials.push({
            id: mat.id,
            count: mat.min && mat.max ? randomInt(mat.min, mat.max) : 1
          });
        }
      }
    }

    // 2. 判断是否掉落装备
    const baseChance = dropTable.isBoss
      ? dropTable.baseDropChance
      : dropTable.baseDropChance * 2.5; // 当前实现：普通怪掉率已上调
    const dropChance = baseChance * (1 + playerMagicFind);
    if (Math.random() < dropChance) {
      // 3. 选择品质
      const quality = this.rollQuality(dropTable.qualityWeights, playerMagicFind);
      
      // 4. 从掉落池随机选一件
      const configId = dropTable.items[Math.floor(Math.random() * dropTable.items.length)];
      
      // 5. 生成装备实例
      const item = EquipmentSystem.generateItem(configId, quality, 'monster');
      drops.push(item);
    }
    
    // 6. 金币掉落
    const gold = Math.floor(dropTable.goldMin + Math.random() * (dropTable.goldMax - dropTable.goldMin));
    
    return { items: drops, materials, gold };
  }
  
  private static rollQuality(weights: Partial<Record<QualityType, number>>, magicFind: number): QualityType {
    // 掉宝率加成提升高品质概率
    const boost = magicFind * 0.5;
    const adjusted = {
      normal:    (weights.normal    ?? 0.50) - boost * 0.3,
      uncommon:  (weights.uncommon  ?? 0.30) - boost * 0.1,
      rare:      (weights.rare      ?? 0.15) + boost * 0.2,
      epic:      (weights.epic      ?? 0.04) + boost * 0.07,
      legendary: (weights.legendary ?? 0.009)+ boost * 0.025,
      mythic:    (weights.mythic    ?? 0.001)+ boost * 0.005
    };
    
    const rand = Math.random();
    let cumulative = 0;
    for (const [quality, weight] of Object.entries(adjusted)) {
      cumulative += Math.max(0, weight);
      if (rand < cumulative) return quality as QualityType;
    }
    return 'normal';
  }
}

interface MaterialDrop {
  id: MaterialId;
  count: number;
}
```

## 7. AI开发提示

### 7.1 实现优先级

1. **P0**：背包增删、穿戴/卸下、属性计算
2. **P1**：套装激活判断、装备强化、掉落生成
3. **P2**：装备鉴定、锁定、排序过滤

### 7.2 关键约束

- 同一部位只能穿一件，穿新装备自动换下旧装备放入背包
- 戒指/手镯属于成对部位，优先穿入空闲的左/右槽位
- 背包满时无法拾取（需提示玩家）
- 套装判断：统计当前穿戴中同setId的件数
- 强化失败：只扣金币，不降级，不破坏
- 武器幸运上限为 7；项链幸运上限为 5；项链幸运 +4/+5 提供额外暴伤/暴击
- 超级祝福油仅通过高级BOSS掉落，不进入普通金币商店
- `allStats` 百分比加成在最终属性时应用（乘法，最后计算）
