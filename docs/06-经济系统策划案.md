# 经济系统策划案

## 1. 系统概述

经济系统管理游戏内所有货币（金币）的产出、消耗和流转，确保7天毕业周期内玩家资源紧张但不断档。当前版本已将药剂补给、武器祝福、项链幸运升级、即时复活、**技能升级**等机制纳入金币消耗闭环，并通过材料掉落（幸运材料 + 技能书页）增强击杀期待值。

## 2. 数据结构

```typescript
interface PlayerWallet {
  playerId: string;
  gold: number;                     // 当前金币
  totalEarned: number;              // 累计获得
  totalSpent: number;               // 累计消耗
  transactions: Transaction[];      // 最近50条流水（用于debug）
}

interface Transaction {
  time: number;
  type: 'earn' | 'spend';
  source: GoldSource;
  amount: number;
  balance: number;                  // 交易后余额
}

type GoldSource =
  | 'combat'          // 打怪
  | 'boss_kill'       // 击杀BOSS
  | 'item_sell'       // 出售装备
  | 'offline_reward'  // 离线收益
  | 'enhance'         // 装备强化
  | 'potion_buy'      // 药剂购买
  | 'blessing_oil'    // 武器祝福油
  | 'necklace_lucky'  // 项链幸运升级
  | 'quick_revive'    // 立即复活
  | 'potion_drop'     // 药剂怪物掉落（减少购买需求）
  | 'skill_upgrade';  // 技能升级
```

  },

  // 装备出售价格（按等级×品质倍率）
  itemSellPrice: (itemLevel: number, quality: QualityType): number => {
    const base = itemLevel * 8;
    const qm = { normal: 1, uncommon: 1.5, rare: 2.5, epic: 5, legendary: 10, mythic: 20 };
    return Math.floor(base * qm[quality]);
  },

  // 离线收益（在线金币/分钟的30%）
  offlineRate: 0.50
};

### 3.1 材料产出补充

- `幸运符石`：从盟重省开始掉落，越高级地图掉率越高，用于项链幸运 `+1 ~ +3`。
- `祝福宝石`：从封魔谷/祖玛后段开始掉落，BOSS 高概率掉落，用于项链幸运 `+3 ~ +5`。
- `超级祝福油`：仅高级BOSS掉落，不进入金币商店，属于高价值稀有资源。
- `技能书页`：所有地图怪物均可掉落，掉率和数量随地图等级递增，BOSS 必掉大量书页，用于技能升级（1→5级）。
- 经济系统虽不直接给材料折现，但需要将材料视为“替代金币门槛”的核心成长资源，与金币共同限制幸运和技能成长速度。

### 4.1 幸运系统经济约束

- 武器幸运成长的金币门槛来自 `祝福油`，成功率只有 50%，因此长期金币消耗稳定。
- `超级祝福油` 不消耗金币，但由高级BOSS限量掉落，承担后期稀缺性设计。
- 项链幸运升级同时消耗 `金币 + 材料 + 成功率`，其中 `+4/+5` 为后期金币深坑。
- `立即复活` 按角色等级收取费用，承担高压地图中的即时容错成本。

### 4.2 技能升级经济约束

- 技能升级消耗 `技能书页 + 金币`，单技能满级需 160 书页 + 40,500 金币。
- 每角色拥有 4~5 个技能，全技能满级总消耗约 **640~800 书页 + 16~20 万金币**。
- 技能升级金币消耗主要集中在 Lv4（8,000）和 Lv5（30,000），形成中后期金币深坑。
- 技能书页从所有地图怪物掉落（前期低概率、后期高概率），BOSS 必掉，鼓励挂机和刷BOSS。
- 技能升级与装备强化、幸运培养共同争夺金币，玩家需要在三者间做优先级决策。

## 5. 核心接口

```typescript
class EconomySystem {
  /** 增加金币 */
  static addGold(playerId: string, amount: number, source: GoldSource): void;

  /** 消耗金币（不足时返回false） */
  static spendGold(playerId: string, amount: number, source: GoldSource): boolean;

  /** 获取当前金币 */
  static getGold(playerId: string): number;

  /** 出售装备 */
  static sellItem(playerId: string, instanceId: string): SellResult;

  /** 购买商店物品 */
  static buyShopItem(playerId: string, itemId: string, quantity: number): BuyResult;

  /** 获取经济统计 */
  static getWallet(playerId: string): PlayerWallet;
}

interface SellResult {
  success: boolean;
  goldGained?: number;
  message?: string;
}

interface BuyResult {
  success: boolean;
  goldCost?: number;
  message?: string;
}
```

**收入/消耗比维持在 1.2~1.4，玩家有积累感但不会暴富。**

> 迭代后需要额外关注：
> 1. 中后期金币是否被 `项链幸运+4/+5` 和 `技能Lv4/Lv5` 同时吸干，导致强化与药剂购买被挤占。
> 2. `超级祝福油` 掉率是否足以支撑运9追求，但又不会完全替代普通祝福油的金币消耗。
> 3. 药剂自动补给在高频挂机场景下是否造成过大的持续金币漏损。
> 4. 技能书页掉率是否让全技能满级在 Day5~6 左右可达成，不能太早（否则无追求）也不能太晚（否则体验割裂）。
> 5. 技能升级金币消耗（全技能约16~20万）需与强化、幸运形成合理竞争，避免某一项被完全忽视。
> 6. 药剂掉落机制（普通怪5%，BOSS 30%）会减少玩家药剂购买支出，需关注是否导致金币积累过快。
> 7. 离线效率已从30%提升至50%，离线装备掉落上限从20件提升至60件，需验证是否影响在线挂机动力。
