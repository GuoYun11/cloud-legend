# 技能系统策划案

## 1. 系统概述

技能系统负责三职业自动战斗中的主动技能、被动技能、召唤物与技能触发逻辑，是连接角色成长、战斗体验与职业差异化的核心系统。

当前版本的目标不是做传统"手动搓技能"，而是做一套**适配放置挂机的自动技能体系**：

- 技能**无需手动释放**，全部通过战斗条件自动触发。
- 技能随等级自动解锁，通过消耗 **技能书页 + 金币** 手动升级（1~5级）。
- 所有职业在 **1级创建角色时即拥有初始技能**。
- 技能与现有的**幸运系统、药剂系统、阵亡/复活、安全区机制**完全兼容。
- 当前实现阶段可先由 `CombatSystem` 承接技能触发，后续再拆分为独立 `SkillSystem.js`。

### 1.1 当前版本同步原则

- 角色等级上限当前为 **70级**，技能解锁节奏围绕 1-70 级设计。
- 法师的 **魔法盾** 解锁等级前置至 16 级，保障法师中前期生存。
- 道士的 **施毒术** 解锁等级前置至 16 级，补足道士持续输出短板。
- 道士将 **灵魂火符** 前置为 **1级初始技能**，替代治愈术承担前期输出定位。
- 道士新增 **无极真气**（60级）作为后期爆发增益技能，补足毕业期战力成长。
- 法师新增 **奥术精通** 终极被动（60级），填补魔法盾前置后的高级技能空缺。
- 技能升级消耗 **技能书页 + 金币**，融入经济闭环，增加杀怪掉落期待值。
- 技能伤害结算沿用战斗系统统一公式，主动输出技能默认吃 `暴击 / 暴击伤害 / 幸运伤害下限 / 减伤`。
- 药剂系统已存在，技能设计不能用"无限回复"破坏补给消耗。
- 阵亡后进入死亡弹窗状态，技能、召唤、持续效果统一暂停。
- 进入 `盟重土城` 等安全区后，不触发战斗技能，不生成召唤物攻击流程。

---

## 2. 设计目标

### 2.1 核心目标

1. 强化职业辨识度，让战士、法师、道士在挂机和打 BOSS 时拥有不同的优势。
2. 保持操作轻量化，所有技能自动释放，玩家只需做职业、装备、地图和资源层面的决策。
3. 技能升级消耗金币和技能书页，融入经济闭环，增加杀怪掉落的期待值。
4. 让技能成为挂机反馈的重要来源，通过日志、飘字、特效提升战斗爽感。

### 2.2 当前版本边界

- **做**：主动技能、被动技能、召唤技能、技能升级（书页+金币）、技能日志、技能冷却、技能优先级。
- **不做**：手动技能栏、技能点树、转职分支、PVP 技能平衡。
- **预留**：未来可增加"觉醒技能""技能符文"扩展，但不进入当前毕业链路。

## 3. 数据结构

### 3.1 技能定义

```typescript
type JobType = 'warrior' | 'mage' | 'taoist';

type SkillType = 'active' | 'passive' | 'summon';

type TriggerType =
  | 'on_attack'        // 普攻时
  | 'on_boss_target'   // 当前目标为BOSS/精英
  | 'on_multi_target'  // 怪物数量达到阈值
  | 'on_low_hp'        // 自身血量低于阈值
  | 'on_map_enter'     // 进入地图时
  | 'on_summon_dead'   // 召唤物死亡时
  | 'always';          // 常驻生效

type SkillEffectType =
  | 'damage'
  | 'heal'
  | 'damage_reduction'
  | 'shield'
  | 'slow'
  | 'summon'
  | 'dot'
  | 'spell_boost'
  | 'buff';

interface SkillDefinition {
  id: string;
  job: JobType;
  name: string;
  type: SkillType;
  unlockLevel: number;
  maxLevel: number;

  trigger: TriggerType;
  cooldownMs?: number;
  manaCost?: number;
  priority: number;           // 越大越先判定

  effect: SkillEffect;
  levelScaling: SkillLevelScaling;
  desc: string;
  logText: string;
}

interface SkillEffect {
  type: SkillEffectType;
  multiplier?: number;        // 输出倍率（1级基础值）
  healRatio?: number;         // 按最大生命恢复比例
  flatHeal?: number;          // 固定恢复值
  targetCount?: number;       // 目标数
  durationMs?: number;        // 持续时间
  value?: number;             // 护盾/减伤/减速/增伤数值
  summonId?: string;          // 召唤物ID
  inheritRatio?: number;      // 召唤物继承比例
  maxStacks?: number;         // DOT层数上限
  mpCostRatio?: number;       // 伤害转MP时的折算比例
}

interface SkillLevelScaling {
  multiplierPerLevel?: number;  // 每级倍率增长（主动输出技能）
  healRatioPerLevel?: number;   // 每级治疗比例增长
  flatHealPerLevel?: number;    // 每级固定治疗增长
  valuePerLevel?: number;       // 每级被动数值增长
  inheritPerLevel?: number;     // 每级召唤继承增长
}
```

技能解锁后初始为 **1级**，通过消耗 **技能书页 + 金币** 手动升级，最高 **5级**。

### 3.5 技能效果缩放公式

```typescript
function getScaledValue(base: number, perLevel: number, skillLevel: number): number {
  return base + perLevel * (skillLevel - 1);
}
```

| 效果类型 | 缩放方式 | 示例（Lv1 → Lv5） |
|---|---|---|
| 主动伤害倍率 | base + multiplierPerLevel × (lv-1) | 攻杀 1.40× → 2.08× |
| 被动减伤/护盾 | base + valuePerLevel × (lv-1) | 护体神盾 8% → 16% |
| DOT伤害比例 | base + valuePerLevel × (lv-1) | 施毒术 25% → 45% |
| 召唤继承比例 | base + inheritPerLevel × (lv-1) | 骷髅 60% → 92% |
| 临时爆发增益 | base + valuePerLevel × (lv-1) | 无极真气 25% → 45% |
| 法伤增幅 | base + valuePerLevel × (lv-1) | 奥术精通 6% → 14% |

## 4. 解锁节奏

### 4.1 各职业技能解锁一览

| 等级 | 战士 | 法师 | 道士 |
|---:|---|---|---|
| **1** | ⚔️ 攻杀剑术（初始） | 🔥 火球术（初始） | 🔥 灵魂火符（初始） |
| **7** | - | - | - |
| **16** | - | 🛡️ 魔法盾 | ☠️ 施毒术 |
| **18** | 🔥 烈火剑法 | - | - |
| **26** | - | - | 💀 召唤骷髅 |
| **30** | - | ❄️ 冰咆哮 | - |
| **45** | ⚡ 逐日剑法 | ☄️ 流星火雨 | 🐉 召唤神兽 |
| **60** | 🛡️ 护体神盾 | ✨ 奥术精通 | ☯️ 无极真气 |

> 战士 4 技能 · 法师 5 技能 · 道士 5 技能

### 4.2 初始技能说明（1级创角即拥有）

所有职业在创建角色时即拥有 **1 个初始技能（Lv1）**，无需额外条件：

| 职业 | 初始技能 | 效果 | 设计意图 |
|---|---|---|---|
| **战士** | 攻杀剑术 | 普攻时概率触发额外斩击（1.4×倍率） | 从第一刀就能感受到战士的肉搏爽感 |
| **法师** | 火球术 | 以低蓝耗法术替代普攻（1.25×倍率） | 建立法师施法手感，蓝耗极低不影响前期 |
| **道士** | 灵魂火符 | 以低蓝耗远程符咒替代普攻（1.30×倍率） | 让道士开局就有明确输出手感，不再靠治疗拖节奏 |

### 4.3 关键节点设计意图

| 等级段 | 设计意图 |
|---|---|
| 1级 | 初始技能确立职业手感，玩家从第一刀就能感受到职业差异 |
| 7级 | 道士没有额外新技能，留出纯火符体验窗口，避免前期技能堆叠过快 |
| 16级 | 法师获得 **魔法盾** 保命；道士获得 **施毒术** 提升持续伤害。两个职业中前期质变点 |
| 18级 | 战士获得 **烈火剑法**，BOSS战爆发质变 |
| 26~30级 | 道士 **召唤骷髅** + 法师 **冰咆哮**，中期清怪效率大幅提升 |
| 45级 | 三职业终极输出技能齐备（逐日/流星/神兽），进入后期爆发阶段 |
| 60级 | 战士 **护体神盾** + 法师 **奥术精通** + 道士 **无极真气**，三职业毕业期全部拿到终局级强化 |

## 5.4 技能蓝耗与职业 HP/MP 特性

### 5.4.1 职业 HP/MP 成长对照

三职业的 HP/MP 成长系数差异化设计，直接决定了技能蓝耗的体感和药剂消耗节奏。

| 属性 | 战士 | 法师 | 道士 | 设计意图 |
|---|---:|---:|---:|---|
| 初始HP | 160 | 80 | 120 | 战士最肉，法师最脆 |
| HP/每级 | 35 | 12 | 22 | 法师HP成长极低，依赖魔法盾 |
| **70级HP** | **2575** | **908** | **1638** | 战士HP是法师的2.8倍 |
| 初始MP | 50 | 130 | 90 | 法师初始蓝量最高 |
| MP/每级 | 4 | 20 | 12 | 法师MP成长最快 |
| **70级MP** | **326** | **1510** | **918** | 法师MP是战士的4.6倍 |
| HP自然恢复/5s | 3 | 1 | 2 | 战士自然回血最高 |
| MP自然恢复/5s | 1 | 4 | 3 | 法师自然回蓝最高 |

### 5.4.2 蓝耗与职业匹配分析

**战士 — 低蓝耗，重平砍**

- 攻杀剑术 **0 MP**，烈火 20 MP / 28s，逐日 45 MP / 52s
- 总蓝耗：约 95 MP/分钟，MP自然恢复 12 MP/分钟
- 净消耗约 83 MP/分钟 → 偶尔补充 MP 药剂即可
- 326 MP 蓝池足够支撑烈火+逐日交替释放，不会因缺蓝卡死输出

**法师 — 高蓝耗，全程施法**

- 火球 8 MP / 2.5s，冰咆哮 28 MP / 18s，流星 60 MP / 75s
- 总蓝耗：约 333 MP/分钟（火球占大头），MP自然恢复 48 MP/分钟
- 净消耗约 285 MP/分钟 → **重度依赖 MP 药剂自动补给**
- 1510 MP 蓝池可支撑约 5 分钟持续施法，避免频繁断蓝
- 魔法盾受击时额外消耗 MP（见下方），进一步加大蓝量压力

**道士 — 中等蓝耗，多技能均摊**

- 火符 6 MP / 3s，施毒 10 MP / 10s，无极真气 35 MP / 45s，召唤一次性 20~40 MP
- 总蓝耗：约 227 MP/分钟（火符占主体，无极真气拉高BOSS战峰值），MP自然恢复 36 MP/分钟
- 净消耗约 191 MP/分钟 → 需要稳定 MP 药剂补给
- 918 MP 蓝池足够维持4~5分钟续航，符合道士"持续消耗 + 爆发窗口"定位

### 5.4.3 魔法盾 MP 消耗机制

魔法盾（被动，16级解锁）在吸收伤害时**额外消耗 MP**，沿用经典传奇设计：

```
受击时 MP 消耗 = 吸收的伤害 × 1.5
```

- Lv1（20%吸收）：受到 100 伤害 → 吸收 20 → 消耗 30 MP
- Lv5（36%吸收）：受到 100 伤害 → 吸收 36 → 消耗 54 MP
- 当 MP 不足以支付吸收时，魔法盾**自动失效**，剩余伤害全部作用于 HP
- 这意味着法师在高压图挂机时，蓝量同时承受"技能施法 + 魔法盾防御"的双重压力，必须依赖 MP 药剂和蓝量管理

> **经济联动**：法师对 MP 药剂的需求远大于其他职业，是最大的药剂金币消耗者。这与战士偏重 HP 药剂、道士两者兼需形成差异化的经济消耗结构。

### 5.4.4 MP 不足时的行为

- 主动技能蓝量不足 → 技能释放失败，回退为普通攻击（0蓝耗）
- 魔法盾蓝量不足 → 盾效果失效，伤害直接作用于 HP
- 召唤技能蓝量不足 → 延迟到蓝量恢复后的下一次判定再补召
- 无极真气蓝量不足 → 跳过本次爆发窗口，道士维持常规火符/施毒/召唤节奏

## 6. 技能触发优先级

### 6.1 同一时刻判定顺序

1. **生存类**：`魔法盾`、`护体神盾`
2. **BOSS爆发类**：`烈火剑法`、`无极真气`
3. **群攻类**：`逐日剑法`、`冰咆哮`、`流星火雨`
4. **持续伤害类**：`施毒术`
5. **常规输出类**：`攻杀剑术`、`火球术`、`灵魂火符`
6. **召唤维护类**：`召唤骷髅`、`召唤神兽`
7. **被动增益类**：`奥术精通`（常驻，不占判定槽位）

### 6.2 具体规则

- 若技能在冷却中、蓝量不足、角色已死亡，则跳过本次判定。
- 若同一 tick 同时满足多个技能条件，只释放优先级最高的一个主动技能。
- 被动技能不占用主动技能释放机会。
- 召唤技能不与治疗技能抢优先级，召唤物阵亡后在下一次可用判定中补召。
- 施毒术判定独立于灵魂火符，同一 tick 可先施毒再释放火符（两者不冲突）。

## 7. 与现有系统的交互

### 7.1 与战斗系统

- 所有主动输出技能统一调用 `CombatCalculator.calcDamage(attacker, defender, skillMultiplier)`。
- 技能天然继承：
  - 幸运伤害下限
  - 暴击率 / 暴击伤害
  - 目标减伤
  - 虚弱带来的属性降低
- 灵魂火符虽为"远程火符"，但在放置挂机中与近战攻击走同一套伤害管线。
- 普通攻击、技能攻击、召唤物攻击都写入战斗日志，但日志最多保留 20 条。

### 7.2 与幸运系统

- 当前版本采用**统一伤害公式**，战士/法师/道士的主动输出技能默认都继承总幸运带来的伤害下限收益。
- 护盾、召唤、DOT、临时增益本身不直接吃幸运倍率。
- 项链幸运 `+4/+5` 带来的暴伤/暴击提升，会间接放大主动输出技能。

### 7.3 与药剂系统

- 技能不替代药剂系统，只负责提供职业特色续航。
- MP 不足时技能释放失败，角色继续执行普通攻击或等待下次判定。
- 自动补给继续按既定阈值工作，和技能系统互补。
- 推荐实现顺序：
  1. 技能判定
  2. 扣除 MP / 执行技能
  3. tick 结束前触发药剂自动补给

### 7.4 与阵亡/复活系统

- 角色阵亡后：
  - 所有主动技能立刻停止
  - 所有临时技能效果暂停/清除
  - 召唤物立即消失
- 角色复活后：
  - 被动技能立即恢复
  - 主动技能重新进入可判定状态
  - 道士召唤物按正常逻辑重新补召
- 虚弱状态作用于角色最终属性，因此会同步影响技能伤害、治疗量和召唤物强度。

### 7.5 与地图/安全区系统

- 在 `盟重土城` 等安全区中，不执行技能判定。
- 进入普通地图时，重新初始化地图技能上下文。
- 道士的召唤技能在进图时优先判定，保证进入战斗后能尽快有宝宝出场。

### 7.6 与经济系统

- 技能升级是中后期重要的 **金币消耗口**（单角色全技能满级约 16~20 万金币）。
- 技能书页作为通用掉落材料，提升了每次击杀的收益感和期待值。
- 技能升级与装备强化、幸运培养、药剂购买共同竞争金币资源，形成多向博弈：
  - 前期：药剂 vs 技能（生存 vs 成长）
  - 中期：强化 vs 技能（装备 vs 技能）
  - 后期：幸运 vs 技能（极限优化阶段）
- 技能书页需纳入 `MATERIAL_CONFIG` 和 `DropConfig` 中统一管理。

## 8. 核心接口

```typescript
class SkillSystem {
  /** 初始化技能状态（创角时调用） */
  static init(playerId: string, job: JobType): void;

  /** 根据玩家等级刷新已解锁技能列表 */
  static refreshUnlockedSkills(playerId: string, level: number): string[];

  /** 获取当前技能运行状态 */
  static getState(playerId: string): PlayerSkillState;

  /** 每次战斗tick时调用，尝试触发技能 */
  static tick(playerId: string, ctx: SkillTriggerContext): SkillCastResult[];

  /** 地图切换后调用，处理召唤、进图类技能 */
  static onEnterMap(playerId: string, mapId: string): void;

  /** 阵亡后调用，清理召唤物和临时技能状态 */
  static onPlayerDead(playerId: string): void;

  /** 复活后调用，重新建立技能运行环境 */
  static onPlayerRevive(playerId: string): void;

  /** 升级技能（消耗技能书页 + 金币） */
  static upgradeSkill(playerId: string, skillId: string): SkillUpgradeResult;

  /** 查询指定技能的下一级升级消耗 */
  static getUpgradeCost(skillId: string, currentLevel: number): SkillUpgradeCost | null;

  /** 获取技能存档数据 */
  static getSkillsData(playerId: string): SkillSaveData;
}

interface SkillCastResult {
  success: boolean;
  skillId: string;
  skillName: string;
  damage?: number;
  heal?: number;
  targets?: number;
  reason?: string;
}

interface SkillUpgradeResult {
  success: boolean;
  skillId: string;
  newLevel?: number;
  reason?: string;         // 失败原因：'max_level' | 'not_enough_gold' | 'not_enough_pages'
}

interface SkillSaveData {
  skillLevels: Record<string, number>;
}
```

## 10. AI开发提示

### 10.1 实现优先级

1. **P0**：技能配置表（含 levelScaling）、解锁规则、升级系统（书页+金币）、战斗中自动触发、技能日志
2. **P1**：道士召唤物、被动技能、技能冷却状态管理、技能书页掉落配置
3. **P2**：技能面板UI、技能特效、离线技能收益折算

### 10.2 关键约束

- 技能升级消耗的 `技能书页` 需在 `MATERIAL_CONFIG` 中注册，并在 `DropConfig` 中配置各地图掉落。
- 技能升级消耗金币需通过 `EconomySystem.spendGold()` 统一记录，`GoldSource` 新增 `'skill_upgrade'`。
- 技能强度必须围绕当前 `70级封顶` 标定，Lv5 技能不能达到破坏平衡的数值。
- 无极真气要强化道士后期战力，但不能让道士瞬时爆发反超同养成法师的纯输出上限。
- 百分比持续伤害要谨慎，施毒术使用"基于角色道术"的公式，避免 BOSS 数值崩坏。
- 召唤物属性必须吃到玩家当前最终属性快照，装备/幸运/虚弱/地图压制都间接影响召唤物。
- 若技能尚未独立拆分，`CombatSystem` 中必须控制好技能判定顺序，避免与普攻/药剂/阵亡逻辑冲突。
- 存档系统需新增 `SKILLS: 'yctl_skills'` 存储键，保存 `skillLevels` 数据。
- 存档迁移需处理旧版（无 `skills` 字段）→ 新版的兼容：默认所有已解锁技能为 Lv1。
