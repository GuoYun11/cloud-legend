## 二、技术栈

```
前端（主要）
├── HTML5 + CSS3 + Vanilla JS（无框架，减少包体）
├── UI 框架：无（手写，基于设计系统 design-tokens.css）
├── 动效：CSS Animation + requestAnimationFrame
└── 音效：Web Audio API

数据存储
├── 主存档：localStorage（JSON 序列化）
└── 云备份（可选）：微信云开发 / 腾讯云 COS

构建工具（可选）
└── Vite + TypeScript（开发提效，最终输出 ES6 bundle）
```

## 五、启动流程

```javascript
// main.js - 游戏启动入口
async function initGame() {
  // 1. 加载存档
  const saveData = SaveSystem.loadAll();
  
  if (!saveData) {
    // 2a. 新用户：进入创建角色流程
    UIManager.showCharacterCreate();
  } else {
    // 2b. 老用户：恢复游戏状态
    await restoreGameState(saveData);
    
    // 3. 结算离线收益
    const offlineReward = OfflineCombatSystem.settleOfflineRewards(saveData.player.id);
    if (offlineReward.minutes > 0) {
      UIManager.showOfflineRewardDialog(offlineReward);
    }
    
    // 4. 恢复挂机
    const lastMapId = saveData.combatState?.mapId ?? 'map_biqi';
    CombatSystem.startCombat(saveData.player.id, lastMapId);
    
    // 5. 启动自动存档
    SaveSystem.startAutoSave(saveData.player.id);
    
    // 6. 显示主界面
    UIManager.showMainPage();
  }
}

initGame();
```

## 八、开发优先级

### 阶段一（P0 核心闭环，约 3-5 天）

- [ ] SaveSystem：存取所有模块数据
- [ ] PlayerSystem：创建角色、经验升级、属性计算
- [ ] EquipmentSystem：背包、穿戴、属性加成
- [ ] CombatSystem：挂机循环（伤害/经验/金币/掉落）
- [ ] EconomySystem：金币增减
- [ ] 主界面UI：角色信息、战斗日志、HP条

### 阶段二（P1 完整玩法，约 3-5 天）

- [ ] MapSystem：地图解锁、切换、BOSS刷新
- [ ] ZodiacSystem：生肖品质掉落收集、套装加成
- [ ] OfflineCombatSystem：离线收益结算
- [ ] 装备UI：穿戴槽、套装显示
- [ ] 背包UI：格子展示、出售
- [ ] 地图UI：解锁列表、BOSS倒计时

### 阶段三（P2 体验优化，约 2-3 天）

- [ ] 生肖UI：12格展示、升级面板
- [ ] 商店UI：物品购买
- [ ] 动效：升级特效、传说掉落特效
- [ ] 音效接入
- [ ] 存档导出/导入
- [ ] 自动出售低品质装备
