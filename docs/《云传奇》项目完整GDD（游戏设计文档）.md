# 《云传奇》项目完整GDD（游戏设计文档）

# 一、项目概述

## 1.1 项目名称

《云传奇》

## 1.2 游戏类型

H5竖版单机、传奇+文字+放置结合，主打情怀还原与轻量化放置体验，无需高精度建模，以文字叙事、数值反馈为核心呈现方式。

## 1.3 核心定位

面向传奇情怀玩家、放置挂机爱好者、碎片化时间用户，打造一款“低门槛、高情怀、强反馈”的轻量化H5单机游戏，既还原传奇核心记忆点，又适配现代玩家“随手玩、轻松成长”的需求，实现“情怀不褪色，放置不无聊”的核心体验。

## 1.4 三大设计支柱（核心原则）

- **支柱一：情怀还原×现代表达**：高度还原传奇核心记忆点（战士-法师-道士职业三角、赤月洞窟等经典地图、装备掉落、怪物图鉴），摒弃高精度3D建模，以富有画面感的文字叙事、简洁UI、高频数值反馈传递情怀，适配文字放置风格。

- **支柱二：放置为体×传奇为魂**：以放置机制（自动化战斗、离线收益、指数级成长）为骨架，保障低门槛和高留存；以传奇元素（职业、装备、怪物、地图）为血肉，提供明确目标和情感驱动力，实现两者深度融合，而非简单叠加。

- **支柱三：确定性成长×随机惊喜**：以放置游戏的“时间必然带来变强”提供确定性成长，给予玩家安全感；以传奇游戏的随机装备掉落、极品触发提供惊喜感，形成“稳定期待+偶发惊喜”的双重驱动，提升玩家粘性。

## 1.5 目标用户

1. 核心用户：25-45岁，有传奇游戏情怀，曾玩过《热血传奇》等经典传奇作品，时间碎片化，追求轻松体验，不喜欢复杂操作的玩家。

2. 次要用户：放置挂机游戏爱好者，喜欢轻量化、低门槛游戏，注重数值成长和随机惊喜，偏好H5单机无需联网、随时可玩的特性。

3. 潜在用户：对传奇IP感兴趣，但不愿投入大量时间和精力玩传统传奇，喜欢文字类游戏的年轻玩家。

## 1.6 核心玩法亮点

- 文字化情怀还原：用细腻文字描绘传奇场景、怪物、装备，搭配简洁图标，唤醒情怀，避免复杂建模，适配H5轻量化特性。

- 全自动化放置：无需手动操作，自动打怪、自动捡取、自动合成装备，离线也能持续获得收益，适配碎片化时间。

- 双重成长驱动：确定性的等级、属性、装备进阶，搭配随机的极品装备、稀有道具掉落，兼顾安全感与惊喜感。

- 竖版H5单机适配：界面简洁，操作便捷，无需下载安装，打开浏览器即可游玩，单机模式无需联网，随时随地可玩。

- 丰富收集体系：涵盖装备、怪物、地图、成就等收集内容，贴合传奇情怀，提升玩家长期留存。

# 二、世界观与剧情设计

## 2.1 世界观设定

沿用传奇经典世界观，简化复杂剧情，聚焦“玛法大陆的重生与守护”：玛法大陆曾因邪恶势力入侵而覆灭，玩家作为被远古神脉唤醒的勇士，肩负起重建大陆、驱逐邪恶的使命，从新手村出发，逐步探索祖玛寺庙、赤月峡谷等经典地图，击败各类怪物BOSS，集齐传奇装备，最终成为玛法大陆的守护者。

世界观核心：回归传奇本质，弱化复杂剧情，强化“探索-打怪-变强-守护”的核心逻辑，用文字片段串联场景与剧情，让玩家在放置成长中感受玛法大陆的情怀与温度，避免冗长叙事带来的阅读负担，每段文字控制在100字以内，简洁易懂且富有画面感。

## 2.2 剧情呈现方式

适配文字放置风格，采用“场景触发式文字叙事”，无需强制观看，玩家可自主选择查看：

- 进入新地图时，弹出简短文字描述（如“赤月峡谷深处，瘴气弥漫，双头金刚的咆哮响彻山谷，传说中法神套装就藏在恶魔祭坛之中”），搭配简洁图标，快速营造场景氛围。

- 击败关键BOSS（如赤月恶魔、魔龙教主）后，解锁剧情片段，讲述BOSS的背景与玛法大陆的过往，唤醒情怀。

- 达成特定成就（如集齐圣战套装、通关所有地图）时，解锁隐藏剧情，补充世界观细节，提升成就感。

## 2.3 核心剧情节点

1. 新手阶段：玩家在新手村苏醒，接受村长委托，击败鸡、鹿等基础怪物，了解游戏玩法，获得第一套基础装备，开启玛法大陆探索之旅。

2. 中期阶段：探索祖玛寺庙、封魔谷等地图，击败祖玛教主等BOSS，收集沃玛、祖玛套装，解锁职业技能，逐步了解玛法大陆的邪恶势力来源。

3. 后期阶段：挑战赤月峡谷、魔龙岭等高级地图，击败赤月恶魔、魔龙教主等顶级BOSS，收集圣战、法神、天尊等套装，解锁四格配饰、生肖装备，逐步变强。

4. 终极阶段：集齐王者/狐月/天龙等毕业装备，解锁所有地图和怪物图鉴，完成狐月秘境等高阶挑战，逐步成为玛法大陆的守护者。

# 三、核心系统设计

## 3.1 职业系统（情怀还原+差异化适配）

还原传奇经典三职业，保持职业三角平衡，适配放置玩法，简化操作，突出数值差异，让玩家根据偏好选择，无需复杂操作即可体验职业特色。

|职业|核心定位|核心技能（当前版本）|成长特性|
|---|---|---|---|
|战士|近战肉盾 + 单体爆发，适合抗伤与打硬目标|攻杀剑术(1)、烈火剑法(18)、逐日剑法(45)、护体神盾(60)|生命、防御、物攻成长最高，后期强在站桩与BOSS爆发|
|法师|远程AOE输出，清怪效率最高，但生存脆|火球术(1)、魔法盾(16)、冰咆哮(30)、流星火雨(45)、奥术精通(60)|魔攻与清怪效率最高，魔法盾前置保命，依赖蓝量管理|
|道士|消耗 + 召唤 + 后期爆发，适合长线挂机|灵魂火符(1)、施毒术(16)、召唤骷髅(26)、召唤神兽(45)、无极真气(60)|属性均衡，前期靠火符破局，中后期靠召唤与爆发窗口维持强度|
补充说明：职业选择在新手阶段完成，一旦选择不可更改；当前版本 `70级封顶`，技能随等级自动解锁，通过消耗 **技能书页 + 金币** 手动升级（1~5级）。

## 3.1.1 技能系统（自动触发 + 手动升级 + 职业差异化）

技能系统采用“**自动解锁、自动触发、手动升级**”的设计：技能随等级自动解锁，战斗中自动触发，玩家通过消耗**技能书页+金币**手动升级技能等级（1~5级）。所有职业在**1级创建角色时即拥有初始技能**。

> 详细配置、触发优先级、升级消耗与接口设计，见 `docs/08-技能系统策划案.md`。

### 3.1.1.1 当前版本核心规则

- 当前版本 `70级封顶`，技能解锁节奏围绕 `1-70级` 设计。
- 技能**无需手动释放**，全部通过战斗条件自动触发。
- 技能随等级自动解锁（初始Lv1），通过消耗**技能书页+金币**手动升级至Lv5。
- 主动输出技能统一走战斗伤害公式，默认吃到暴击率/暴击伤害、幸运伤害下限、目标减伤、虚弱减益。
- 进入 `盟重土城` 等安全区后，不触发技能；角色阵亡后，技能与召唤物统一暂停。

### 3.1.1.4 交互与平衡约束

- 技能系统与药剂系统互补，不允许任何职业通过技能跳过补给成本。
- 道士偏向“持续消耗 + 召唤 + 爆发窗口”，法师偏向“清怪效率”，战士偏向“单体压制与生存”。
- 施毒术按角色道术比例结算，不使用百分比扣血方案，避免BOSS数值崩坏。
- 技能升级消耗金币（全技能约16~20万），与装备强化、幸运培养形成多向资源博弈。
- 技能书页作为通用掉落材料，增加杀怪期待值，融入经济闭环。
- 输出技能默认继承幸运系统收益，项链幸运 +4/+5 的暴伤、暴击也会放大技能表现。

## 3.2 放置挂机系统（核心骨架）

以“全自动化+离线收益”为核心，简化操作，保障低门槛，让玩家无需手动干预，即可实现持续成长，贴合碎片化时间需求，同时兼顾传奇打怪的核心体验。

### 3.2.1 核心功能

- 自动战斗：玩家选择地图后，角色自动前往该地图，自动攻击范围内的怪物，无需手动点击，战斗过程以文字提示呈现（如“你攻击了白野猪，造成XXX点伤害”“白野猪掉落了沃玛手镯”）。

- 自动拾取：怪物掉落的装备、道具、金币，角色自动拾取，无需手动操作，拾取后弹出简短文字提示，提升反馈感；可设置“自动拾取过滤”，选择只拾取指定品质以上的装备，避免背包冗余。

- 离线收益：玩家退出游戏后，角色继续在当前地图挂机，持续获得经验、金币、装备，离线时间越长，收益越多（设置上限，避免过度差距）；再次登录时，弹出离线收益汇总提示，点击即可领取，给予玩家明确的成长反馈。

- 挂机设置：支持自定义挂机地图、挂机目标（如“优先攻击BOSS”“优先清怪”）、拾取规则，适配不同玩家需求；支持“一键挂机”，新手玩家可直接默认设置，快速上手。

### 3.2.2 成长节奏设计（指数级成长）

贴合放置游戏核心，采用“前快后稳”的成长模式：前期快速解锁职业手感，中期逐步进入装备与技能成型期，后期围绕毕业地图、幸运系统和高阶套装展开追求。

- 前期（1-20级）：每10分钟可提升5-10级，1级即拥有初始技能（攻杀/火球/治愈），7级道士解锁灵魂火符，16级法师/道士分别获得魔法盾/施毒术，快速建立职业认知。

- 中期（21-45级）：成长速度略放缓，每20-30分钟提升4-8级，逐步解锁中期技能（烈火/冰咆/骷髅）、祖玛/赤月级地图与成型散件，技能升级消耗技能书页+金币。

- 后期（46-60级）：成长进一步趋稳，每30-40分钟提升3-6级，解锁终极技能（逐日/流星/神兽）、终极被动（护体神盾/奥术精通）、四格配饰、生肖系统和高压BOSS挑战。

- 毕业期（61-70级）：等级提升明显放缓，核心目标转向高阶地图、幸运培养、技能满级（Lv5）、星王/王者/天龙等毕业装备，以及高级BOSS的反复挑战。

## 3.3 装备系统（传奇核心+文字呈现）

还原传奇经典装备体系，涵盖常规套装、四格配饰、生肖装备，以文字描述+简洁图标呈现，突出装备进阶感和随机惊喜，贴合“确定性成长+随机惊喜”的支柱，同时沿用之前整理的装备-地图-BOSS对应关系，保障情怀还原度。

### 3.3.1 装备分类与进阶（确定性成长）

1. 常规套装（核心装备）：按当前版本配置，套装从中期到毕业期的主要路线如下：

    - 过渡阶段：祖玛套装
    - 职业成型：圣战 / 法神 / 天尊
    - 中后期泛用：魔龙 / 雷霆 / 雷炎
    - 毕业前成长：星王 / 卧龙 / 王者 / 狐月
    - 毕业阶段：天龙套装

2. 饰品与四格装备（辅助装备）：当前版本固定为项链、左右戒指、左右手镯、腰带、鞋子，以及四格配饰 `斗笠 / 勋章 / 军鼓 / 符咒`。其中：

    - 左右戒指、左右手镯均为独立槽位，支持成对穿戴
    - 四格装备以线性升级为主，提升经验、攻速、暴击、生存等附加属性
    - 项链额外承担“幸运成长”功能，可通过金币与材料升级幸运值

3. 生肖装备（特殊装备）：12件对应十二生肖，按品质进阶，单件提升属性，套装触发额外效果，进阶路线固定：
       

    - 品质进阶：下品生肖→中品生肖→上品生肖→极品生肖→仙品生肖→传奇生肖（终极生肖装备）

    - 套装效果：6件触发基础套装效果，12件触发终极套装效果，提升角色核心属性和PK、刷怪效率。

### 3.3.2 装备获取（随机惊喜+确定性产出）

结合传奇经典掉落机制与放置挂机特性，实现“确定性产出+随机惊喜”的双重体验，所有装备产出严格对应之前整理的装备-地图-BOSS对应关系，保障情怀还原：

- 确定性产出：
        

    - 基础装备（布衣、木剑、铁装等）：低阶地图小怪掉落，挂机即可稳定获取，保障新手成长。

    - 套装散件：指定地图小怪、BOSS掉落对应阶段的散件与配套部位，玩家通过持续挂机逐步凑齐目标套装，实现确定性成长。

    - 四格配饰、生肖装备（低品质）：对应低阶、中阶地图小怪、精英掉落，持续挂机可稳定获取，逐步进阶。

- 随机惊喜：
       

    - 极品装备：BOSS有概率掉落极品装备（如+10圣战头盔、传奇级斗笠），掉落时弹出醒目文字提示（如“恭喜你！击败赤月恶魔，掉落极品法神戒指！”），提升惊喜感。

    - 稀有道具：BOSS、精英有概率掉落装备强化材料、生肖进阶道具，以及幸运系统所需的 `幸运符石 / 祝福宝石 / 超级祝福油`，提升挂机期待值。

    - 毕业装备：顶级BOSS（如狐月天柱）有概率直接掉落高品质毕业套装或极品散件，形成后期刷图追求。

### 3.3.3 装备强化与合成（确定性成长）

简化养成操作，但保留传奇系数值追求：

- 装备强化：装备强化上限为 `+10`，每级提供约 `8%` 属性成长；强化存在成功率，但失败仅扣金币，不降级、不碎装。

- 幸运培养：
  - 武器可使用 `祝福油 / 超级祝福油` 提升幸运，目标是达成 `运9 = 100%攻击上限伤害`
  - 项链不做独立幸运项链系列，所有项链都可通过 `金币 + 材料 + 成功率` 提升幸运，最高 `+5`
  - 项链幸运 `+4` 提供暴击伤害提升，`+5` 提供暴击率提升

- 四格配饰、生肖装备进阶：通过对应进阶道具逐步提升，不承担主线成长的数值核心，但负责补足中后期属性厚度。

### 3.3.4 装备呈现方式

适配文字放置风格，以“文字描述+简洁图标”呈现，避免复杂建模，突出情怀和数值反馈：

- 装备图标：采用简洁复古风格图标，贴合传奇经典形象，无需高精度设计，适配H5轻量化特性。

- 文字描述：每件装备配有简短文字描述，突出传奇情怀和属性特点（如“圣战头盔：玛法大陆战士的顶级头盔，蕴含强大的防御力量，是征战赤月的必备装备”）。

- 装备穿戴反馈：穿戴装备后，角色属性即时提升，弹出文字提示（如“穿戴圣战套装，物理攻击+1000，防御+800”），给予玩家明确的成长反馈。

## 3.4 地图与怪物系统（情怀还原+放置适配）

还原传奇经典地图和怪物，以文字描述+简洁图标呈现，地图按难度逐步解锁，怪物掉落与装备体系严格对应，适配放置挂机玩法，让玩家在探索中感受传奇情怀。

### 3.4.1 地图设计（情怀还原+逐步解锁）

1. 地图分类与解锁条件：当前版本共有 **16张地图（含盟重土城安全区）**，按 1-70 级节奏逐步解锁：

    - 安全区：盟重土城（默认开放），不产出经验金币，不触发战斗，用于回城、脱战和阵亡恢复。

    - 新手阶段：比奇省、盟重省、石墓阵、封魔谷（1-15级），掉落基础装备、祖玛前置散件与低级幸运材料。

    - 中期阶段：祖玛寺庙1-4层、祖玛寺庙5-7层、赤月峡谷（20-30级），进入职业套装成型期。

    - 中后期阶段：困惑殿堂、地狱烈焰、魔龙岭、魔龙血域（40-50级），开始承接更高金币收益、生肖成长和高阶材料掉落。

    - 高阶阶段：雷炎洞穴、卧龙山庄、狐月山、狐月秘境（55-70级），作为毕业地图与顶级BOSS挑战区。

2. 地图呈现方式：以文字描述+地图缩略图呈现，进入地图后，弹出场景描述（如“祖玛寺庙5层，石柱林立，祖玛卫士手持巨斧，守护着祖玛教主的宫殿，空气中弥漫着古老的气息”），无需复杂场景建模，贴合文字放置风格。

3. 地图玩法：每个战斗地图都配置普通怪与地图BOSS，BOSS定时刷新；玩家阵亡后可选择立即复活或回到 `盟重土城`，安全区承担当前版本的恢复与缓冲职能。

### 3.4.2 怪物系统（情怀还原+掉落对应）

- 怪物分类：按难度分为基础怪物、精英怪物、BOSS怪物，还原传奇经典怪物形象，以文字描述+简洁图标呈现，标注怪物等级、血量、掉落物品：

    - 基础怪物：鸡、鹿、稻草人、祖玛卫士等，血量低、伤害低，掉落基础装备、金币，适合新手挂机。

    - 精英怪物：极品祖玛卫士、精英白野猪等，血量中等、伤害中等，掉落中阶装备、套装散件与稀有材料，是中期挂机的主要目标。

    - BOSS怪物：祖玛教主、赤月恶魔、魔龙教主等，血量高、伤害高，掉落高阶装备、稀有道具、极品装备，是后期玩家的核心挑战目标。

- 怪物掉落：严格对应之前整理的“装备-地图-BOSS”对应关系，确保情怀还原，同时兼顾放置挂机的随机性和确定性（基础掉落稳定，极品掉落随机）。

- 怪物图鉴：解锁“怪物图鉴”系统，玩家击败怪物后，可解锁该怪物的图鉴，包含怪物描述、掉落物品、背景故事，收集满图鉴可获得额外奖励，提升收集乐趣，贴合传奇情怀。

## 3.5 生肖系统（特殊收集+属性提升）

还原传奇生肖玩法，适配放置挂机特性，以“收集+套装加成”为核心，提升角色属性，增加收集乐趣，贴合“确定性成长+随机惊喜”的支柱。

- 生肖装备：12件对应十二生肖（子鼠、丑牛、寅虎、卯兔、辰龙、巳蛇、午马、未羊、申猴、酉鸡、戌狗、亥猪），每件生肖装备有独立属性，按品质进阶（下品→中品→上品→极品→仙品→传奇）。

- 获取方式：低品质生肖装备可通过中阶、高阶地图小怪、精英掉落；高品质生肖装备可通过BOSS掉落、生肖进阶获得；传奇生肖装备主要由高阶BOSS与终极地图掉落产出，保持稀缺性与长期追求。

围绕传奇情怀和放置成长，设计丰富的成就和收集内容，给予玩家额外奖励，提升长期留存，同时唤醒玩家的传奇情怀。

### 3.6.1 成就系统

    - 成长类：等级提升、技能解锁、属性突破等（如“等级达到70级”“解锁终极技能”）。

    - 装备类：集齐指定套装、强化装备到指定等级、获得极品装备等（如“集齐圣战套装”“强化装备到+10”“获得传奇斗笠”）。

简化货币和道具体系，适配放置挂机玩法，无需复杂交易，所有道具均可通过挂机、BOSS掉落获得，保障低门槛，同时避免氪金压力（单机无氪金）。

- 货币用途：装备强化、技能升级、药剂补给、祝福油、项链幸运升级、生肖升级、立即复活等，整体围绕“挂机产出 -> 成长消耗 -> 再次挂机提升”闭环设计。

    - 续航道具：回血、回蓝道具，可通过小怪掉落，挂机时自动使用，保障角色生存。

    - 稀有道具：用于限定称号、收藏奖励与后续扩展玩法预留，可通过BOSS、成就获得，提升稀缺性和惊喜感。

# 五、技术实现（H5竖版单机）

## 5.1 技术栈选择

- 前端：HTML5 + CSS3 + JavaScript（ES6 模块），以原生实现为主，不引入重型框架；通过轻量 DOM 渲染、定时循环与响应式样式保障竖版 H5 的加载速度和流畅度。

- 后端：当前版本无需后端服务器（单机游戏），核心数据全部存储在本地 `localStorage`，保障玩家离线可玩、刷新后进度保留。

- 资源：采用轻量化资源（简洁图标、文字描述），避免高精度图片和复杂动画，保障H5加载速度，适配不同手机机型；优化音频播放，解决iOS、Android平台音频播放兼容问题，采用DOM音频预加载（先播放再暂停）的方式，确保音频正常播放。

## 5.2 核心技术实现要点

- 放置挂机机制：通过JavaScript定时器实现自动战斗、自动拾取、离线收益计算，离线收益按退出时间累计，再次登录时自动结算，设置收益上限，避免过度差距。

- 本地数据存储：将玩家角色信息、装备、金币、地图、生肖、药剂、幸运材料、技能等级、战斗状态等数据存储在 `localStorage`，确保玩家退出游戏后数据不丢失。

- 装备与怪物掉落机制：基于随机数算法，实现装备掉落的随机性（极品装备低概率，基础装备高概率），同时严格对应“装备-地图-BOSS”对应关系，保障情怀还原。

- 界面适配：采用响应式布局，适配不同手机屏幕尺寸，优化竖屏显示效果；解决H5竖屏转横屏的适配问题，避免出现界面旋转异常、显示不全等问题。

- 性能优化：简化界面元素和动画，减少资源加载量，保障游戏流畅运行；优化代码，避免卡顿，适配中低端手机机型。

## 5.3 兼容性要求

- 浏览器兼容：支持主流手机浏览器（微信浏览器、QQ浏览器、Chrome、Safari等），版本要求在近3个版本内，确保游戏正常运行。

- 机型兼容：适配主流手机机型（安卓、iOS），屏幕尺寸涵盖4.7-6.7英寸，确保界面显示正常、操作便捷；优化低配置机型运行效果，避免卡顿。

- 网络兼容：单机游戏，无需联网即可游玩，联网状态下可优化加载速度（如加载最新资源），断网后不影响游戏正常运行。

# 六、数值设计（核心平衡+双重驱动）
