# 生肖系统策划案

## 1. 系统概述

生肖系统是《云传奇》的特色养成模块。玩家通过击杀怪物直接获得不同品质的完整生肖，品质越高属性越强，集齐套装可激活强力套装效果。生肖装备分为下品/中品/上品/极品/仙品/传奇六档品质。

## 2. 数据结构

```typescript
// 玩家生肖进度
interface ZodiacProgress {
  playerId: string;
  animals: ZodiacAnimalData[];     // 12个生肖数据
  totalStats: ZodiacTotalStats;    // 汇总属性（缓存，随掉落与品质替换刷新）
}

// 单个生肖数据
interface ZodiacAnimalData {
  animalId: number;                // 1-12
  animalName: string;
  quality: ZodiacQuality;          // 当前装备的品质
  level: number;                   // 当前等级 (0=未获得, 1-15)
  isEquipped: boolean;             // 是否已激活
  totalStats: AnimalStatValue;     // 当前提供的属性
}

// 生肖品质
type ZodiacQuality = 'none' | 'low' | 'mid' | 'high' | 'elite' | 'immortal' | 'legendary';

// 生肖属性值
interface AnimalStatValue {
  statType: string;                // 属性类型
  value: number;                   // 属性值
}

// 生肖汇总属性
interface ZodiacTotalStats {
  attack: number;
  defense: number;
  maxHp: number;
  critRate: number;
  critDamage: number;
  dodgeRate: number;
  attackSpeed: number;
  hpRegen: number;
  expBonus: number;
  goldBonus: number;
  magicFind: number;
  allStatsPercent: number;         // 龙的全属性加成
  // 套装加成
  setBonus3?: boolean;
  setBonus6?: boolean;
  setBonus9?: boolean;
  setBonus12?: boolean;
}
```

### 3.2 品质掉落权重

怪物击杀时按地图等级分层，roll到的品质权重如下：

| 地图等级档位 | 对应地图 | 下品 | 中品 | 上品 | 极品 | 仙品 | 传奇 |
|---|---|---|---|---|---|---|---|
| low (Lv1-19) | 比奇郊外~封魔谷 | 70 | 25 | 5 | 0 | 0 | 0 |
| mid (Lv20-34) | 祖玛寺庙 | 40 | 40 | 15 | 5 | 0 | 0 |
| high (Lv35-49) | 赤月峡谷~魔龙岭 | 15 | 35 | 30 | 15 | 5 | 0 |
| elite (Lv50-64) | 魔龙血域~卧龙山庄 | 5 | 20 | 30 | 25 | 15 | 5 |
| top (Lv65-70) | 狐月山~狐月秘境 | 0 | 10 | 20 | 30 | 25 | 15 |

掉落概率：普通怪 3%，BOSS 25%。

### 3.3 品质等级映射

获得生肖后，品质直接映射为内部等级用于属性计算：

| 品质 | 内部等级 | 属性公式 |
|---|---|---|
| 下品 (low) | Lv1 | base + growth × 0 |
| 中品 (mid) | Lv3 | base + growth × 2 |
| 上品 (high) | Lv5 | base + growth × 4 |
| 极品 (elite) | Lv8 | base + growth × 7 |
| 仙品 (immortal) | Lv11 | base + growth × 10 |
| 传奇 (legendary) | Lv15 | base + growth × 14 |

获得更高品质的同一生肖时，自动替换为高品质版本。低品质不会覆盖高品质。

## 4. 核心接口

```typescript
class ZodiacSystem {
  /** 获取玩家全部生肖进度 */
  static getProgress(playerId: string): ZodiacProgress;

  /** 获取单个生肖数据 */
  static getAnimal(playerId: string, animalId: number): ZodiacAnimalData;

  /**
   * 怪物掉落完整生肖时调用；若新品质高于已有品质则替换为高品质版本，否则忽略。
   */
  static addZodiac(playerId: string, animalId: number, quality: ZodiacQuality): void;

  /** 替换生肖品质（内部调用 addZodiac） */
  static replaceQuality(playerId: string, animalId: number, newQuality: ZodiacQuality): boolean;

  /** 计算生肖提供的所有属性（含套装加成） */
  static getTotalStats(playerId: string): ZodiacTotalStats;

  /** 获取当前套装激活状态 */
  static getSetBonusStatus(playerId: string): ZodiacSetStatus;
}

interface ZodiacSetStatus {
  equippedCount: number;           // 当前激活数量
  activeBonuses: string[];         // 已激活套装加成描述
  nextBonus?: string;              // 下一个套装加成说明
  missingAnimals: string[];        // 还缺哪些生肖
}
```

## 5. 属性计算逻辑

```typescript
class ZodiacCalculator {
  static calcTotalStats(progress: ZodiacProgress): ZodiacTotalStats {
    const totals: ZodiacTotalStats = {
      attack: 0, defense: 0, maxHp: 0,
      critRate: 0, critDamage: 0, dodgeRate: 0,
      attackSpeed: 0, hpRegen: 0, expBonus: 0,
      goldBonus: 0, magicFind: 0, allStatsPercent: 0
    };

    let equippedCount = 0;

    for (const animal of progress.animals) {
      if (animal.level === 0 || !animal.isEquipped) continue;

      equippedCount++;
      const config = ZODIAC_ANIMAL_CONFIG[animal.animalId - 1];
      const base  = config.baseValue[animal.quality];
      const growth = config.growthPerLevel[animal.quality];
      const value = base + growth * (animal.level - 1);

      (totals as any)[config.statType] += value;
    }

    // 套装加成
    for (const bonus of ZODIAC_SET_BONUS) {
      if (equippedCount >= bonus.count) {
        Object.entries(bonus.bonus).forEach(([k, v]) => {
          if (k === 'allStatsAdd') {
            totals.allStatsPercent += v as number;
          } else {
            (totals as any)[k] = ((totals as any)[k] ?? 0) + v;
          }
        });
      }
    }

    return totals;
  }
}
```

## 6. AI开发提示

### 6.1 实现优先级

1. **P0**：生肖品质掉落、属性汇总计算
2. **P1**：套装激活判断、品质替换
3. **P2**：生肖图鉴展示、收集进度动画

### 6.2 关键约束

- 每种生肖只激活一个；同一生肖只保留最高品质，低品质不会覆盖高品质
- 生肖通过怪物掉落获得，无碎片/合成机制
- 套装加成按激活数量叠加（满足3件同时满足3/6/9/12件时全部生效）
- `allStatsPercent` 在最终属性计算时做乘法加成，最后应用
