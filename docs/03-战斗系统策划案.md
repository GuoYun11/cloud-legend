# 战斗系统策划案

## 1. 系统概述

战斗系统是《云传奇》的核心玩法。采用**自动挂机**模式，玩家角色持续与怪物战斗，周期性结算伤害、经验、金币、装备与材料掉落。当前版本已接入自动补给药剂、幸运伤害、药剂怪物掉落、生肖怪物掉落、阵亡弹窗复活、盟重土城安全区与虚弱惩罚机制。

---

## 2. 战斗流程

```
进入地图
  └─> 开始挂机循环 (TICK = 1秒)
        ├─ 每TICK：角色对怪物造成伤害
        ├─ 每TICK：怪物对角色造成伤害
        ├─ 每1秒：检测自动补给药剂（回血/回蓝）
        ├─ 每5秒：角色生命/魔法自然恢复
        ├─ 怪物死亡 → 结算奖励 → 刷新下一只
        └─ 角色死亡 → 阵亡弹窗（立即复活/回城/自动复活）
  └─ BOSS战（独立流程）
```

### 2.1 当前版本补充规则

- 安全区地图不进入战斗循环，进入后仅保留挂机状态展示和功能入口。
- 自动补给由独立 `PotionSystem` 执行，默认可设置 HP/MP 阈值与药剂类型。
- 死亡后不会立刻原地拉起，而是进入阵亡状态，等待玩家选择复活方式或倒计时自动复活。
- 连续死亡会叠加 `虚弱`，每层降低 10% 关键属性，持续 30 秒，最多 3 层。

## 4. 伤害计算公式

```typescript
class CombatCalculator {
  /**
   * 计算一次攻击的伤害
   * @param attacker 攻击方属性
   * @param defender 防御方属性
   * @param skillMultiplier 技能倍率（普通攻击=1.0）
   */
  static calcDamage(
    attacker: CombatUnit,
    defender: CombatUnit,
    skillMultiplier: number = 1.0
  ): DamageResult {
    // 1. 闪避判定
    const dodgeRoll = Math.random();
    if (dodgeRoll < defender.dodgeRate) {
      return { damage: 0, isMiss: true, isCrit: false };
    }
    
    // 2. 防御百分比减免 reduction = defense / (defense + 150)，保底为攻击力的10%
    const defK = 150;
    const defReduction = defender.defense / (defender.defense + defK);
    const maxDamage = Math.max(
      attacker.attack * 0.1,
      attacker.attack * (1 - defReduction)
    );
    
    // 3. 幸运修正：运0=70%~100%，运9=100%上限伤害
    const lucky = Math.min(attacker.lucky ?? 0, 9);
    const minRatio = 0.7 + (lucky / 9) * 0.3;
    const variance = 1.0 - minRatio;
    let damage = maxDamage * (minRatio + Math.random() * variance);

    // 4. 技能倍率
    damage *= skillMultiplier;
    
    // 5. 暴击判定
    const critRoll = Math.random();
    const isCrit = critRoll < attacker.critRate;
    if (isCrit) {
      damage *= attacker.critDamage;
    }
    
    // 6. 减伤计算（防御方的减伤比例）
    damage *= (1 - (defender.damageReduction ?? 0));
    
    return {
      damage: Math.max(1, Math.floor(damage)),
      isMiss: false,
      isCrit
    };
  }
  
  /**
   * 计算怪物对玩家的伤害（简化版，无暴击/闪避）
   * 技能系统的 damageReduction 与 magicShield 在返回值基础上另行结算。
   */
  static calcMonsterDamage(monsterAttack: number, playerDefense: number): number {
    const defK = 150;
    const plrDefReduction = playerDefense / (playerDefense + defK);
    let baseDamage = Math.max(monsterAttack * 0.1, monsterAttack * (1 - plrDefReduction));
    baseDamage *= (0.9 + Math.random() * 0.2);
    return Math.max(1, Math.floor(baseDamage));
  }
}

interface DamageResult {
  damage: number;
  isMiss: boolean;
  isCrit: boolean;
}

interface CombatUnit {
  attack: number;
  defense: number;
  lucky?: number;
  critRate: number;
  critDamage: number;
  dodgeRate: number;
  damageReduction?: number;
}
```

怪物对玩家的最终承伤：先由 `calcMonsterDamage` 得到基础伤害，再叠加技能系统的 `damageReduction`、`magicShield` 等减免。

### 4.1 幸运与补给说明

- 角色总幸运由 `武器幸运 + 项链幸运 - 诅咒` 组成，参与普通攻击伤害浮动计算。
- 项链幸运 `+4` 额外提升 `暴击伤害+15%`，`+5` 额外提升 `暴击率+5%`。
- 自动补给每秒检测一次，优先按玩家设定的药剂类型与阈值消耗库存。
- 材料掉落与装备掉落分开结算，材料不会挤占装备掉落机会。

### 4.2 药剂掉落

怪物击杀时有概率直接掉落药剂（独立于装备掉落）：

- 普通怪掉率 5%，BOSS 掉率 30%
- 药剂类型根据怪物等级分层：Lv1-9 小药，Lv10-24 中药，Lv25-44 大药，Lv45+ 极品药
- 掉落数量：普通怪 1-2 瓶，BOSS 2-5 瓶
- HP药和MP药均可掉落
- 药剂冷却：HP药和MP药各有独立的2秒冷却

### 4.3 生肖掉落

怪物击杀时有概率直接掉落完整生肖（独立于装备掉落）：

- 普通怪掉率 3%，BOSS 掉率 25%
- 随机获得12生肖中的一种
- 品质根据地图等级分层（详见生肖系统策划案 §3.2）
- 若获得品质高于已有则自动替换，低品质不覆盖高品质

## 5. 战斗循环（挂机核心逻辑）

```typescript
class CombatSystem {
  private static tickInterval: number = 1000; // 每秒tick一次
  private static timers: Map<string, ReturnType<typeof setInterval>> = new Map();
  
  /** 开始挂机 */
  static startCombat(playerId: string, mapId: string): void {
    const state = this.initCombatState(playerId, mapId);
    
    // 生成第一只怪
    state.currentMonster = MonsterSpawner.spawn(mapId);
    
    const timer = setInterval(() => {
      this.tick(state);
    }, this.tickInterval);
    
    this.timers.set(playerId, timer);
    this.saveCombatState(state);
  }
  
  /** 停止挂机 */
  static stopCombat(playerId: string): void {
    const timer = this.timers.get(playerId);
    if (timer) {
      clearInterval(timer);
      this.timers.delete(playerId);
    }
  }
  
  /** 每秒执行一次 */
  private static tick(state: CombatState): void {
    const now = Date.now();
    const player = PlayerSystem.getPlayerData(state.playerId);
    const playerStats = PlayerSystem.getFinalStats(state.playerId);
    
    if (!state.currentMonster) {
      state.currentMonster = MonsterSpawner.spawn(state.mapId);
      return;
    }
    
    // === 玩家攻击 ===
    const attackInterval = 1000 / playerStats.attackSpeed; // 毫秒
    if (now - state.lastAttackTime >= attackInterval) {
      state.lastAttackTime = now;
      
      const result = CombatCalculator.calcDamage(
        { attack: playerStats.attack, defense: 0, critRate: playerStats.critRate, critDamage: playerStats.critDamage, dodgeRate: 0 },
        { attack: 0, defense: state.currentMonster.defense, critRate: 0, critDamage: 1, dodgeRate: 0 }
      );
      
      if (!result.isMiss) {
        state.currentMonster.currentHp -= result.damage;
      }
      
      // 记录战斗日志
      const logText = result.isMiss
        ? `你攻击${state.currentMonster.name}，未命中`
        : `你攻击${state.currentMonster.name}，造成${result.isCrit ? '【暴击】' : ''}${result.damage}点伤害`;
      
      this.addLog(state, 'player_attack', logText, result.damage, result.isCrit, result.isMiss);
      
      // 怪物死亡
      if (state.currentMonster.currentHp <= 0) {
        this.onMonsterDead(state, player, playerStats);
        return;
      }
    }
    
    // === 怪物攻击 ===
    const monsterAttackInterval = state.currentMonster ? (state.currentMonster.attackSpeed ?? 2) * 1000 : 2000;
    if (now - state.lastMonsterAttackTime >= monsterAttackInterval) {
      state.lastMonsterAttackTime = now;
      
      const monsterDmg = CombatCalculator.calcMonsterDamage(
        state.currentMonster.attack,
        playerStats.defense
      );
      
      PlayerSystem.consumeHp(state.playerId, monsterDmg);
      
      this.addLog(state, 'monster_attack',
        `${state.currentMonster.name}攻击你，造成${monsterDmg}点伤害`,
        monsterDmg
      );
      
      // 玩家死亡检测
      const updatedPlayer = PlayerSystem.getPlayerData(state.playerId);
      if (updatedPlayer.currentHp <= 0) {
        this.onPlayerDead(state, playerStats);
      }
    }
    
    // === 自然恢复（每5秒） ===
    if (now - state.lastRegenTime >= 5000) {
      state.lastRegenTime = now;
      PlayerSystem.restoreHp(state.playerId, playerStats.hpRegen);
      PlayerSystem.restoreMp(state.playerId, playerStats.mpRegen);
    }
    
    this.saveCombatState(state);
  }
  
  /** 怪物死亡处理 */
  private static onMonsterDead(state: CombatState, player: PlayerData, playerStats: FinalStats): void {
    const monster = state.currentMonster!;
    
    // 1. 经验奖励（含加成）
    const expGained = Math.floor(monster.expReward * (1 + playerStats.expBonus));
    const levelResult = PlayerSystem.addExp(state.playerId, expGained);
    
    // 2. 金币奖励（含加成）
    const goldBase = monster.goldMin + Math.floor(Math.random() * (monster.goldMax - monster.goldMin));
    const goldGained = Math.floor(goldBase * (1 + playerStats.goldBonus));
    EconomySystem.addGold(state.playerId, goldGained);
    
    // 3. 装备掉落 + 材料掉落
    const dropResult = DropSystem.rollDrop(state.mapId, monster.configId, playerStats.magicFind);
    if (dropResult.items.length > 0) {
      dropResult.items.forEach(item => {
        EquipmentSystem.addToInventory(state.playerId, item);
        this.addLog(state, 'item_drop', `获得【${item.configId}】`, undefined);
      });
      state.sessionStats.itemsDropped += dropResult.items.length;
    }
    if (dropResult.materials.length > 0) {
      dropResult.materials.forEach(mat => {
        EquipmentSystem.addMaterial(state.playerId, mat.id, mat.count);
        this.addLog(state, 'drop', `获得材料 ${mat.id} ×${mat.count}`);
      });
    }
    
    // 4. 更新统计
    state.sessionStats.kills++;
    state.sessionStats.expGained += expGained;
    state.sessionStats.goldGained += goldGained;
    if (monster.isBoss) state.sessionStats.bossKills++;
    
    // 5. 升级提示
    if (levelResult.leveledUp) {
      this.addLog(state, 'level_up', `🎉 恭喜升级到 ${levelResult.newLevel} 级！`);
    }
    
    this.addLog(state, 'player_attack',
      `击败${monster.name}，获得经验${expGained}，金币${goldGained}`
    );
    
    // 6. 刷新下一只怪
    state.currentMonster = MonsterSpawner.spawn(state.mapId);
  }
  
  /** 玩家死亡处理 */
  private static onPlayerDead(state: CombatState, playerStats: FinalStats): void {
    state.isDead = true;
    state.deathStreak++;
    state.currentMonster = null;
    state.reviveAt = Date.now() + 5000;
    this.addLog(state, 'death', '你已阵亡，等待复活');
  }
  
  private static addLog(state: CombatState, type: BattleLogEntry['type'], text: string, value?: number, isCrit?: boolean, isMiss?: boolean): void {
    state.battleLog.unshift({ time: Date.now(), type, text, value, isCrit, isMiss });
    if (state.battleLog.length > 20) state.battleLog.pop();
  }
  
  private static initCombatState(playerId: string, mapId: string): CombatState {
    return {
      playerId, mapId,
      isActive: true, isPaused: false, isDead: false,
      currentMonster: null,
      lastAttackTime: 0,
      lastMonsterAttackTime: 0,
      lastRegenTime: Date.now(),
      reviveAt: 0,
      deathStreak: 0,
      weakStacks: 0,
      weakExpiresAt: 0,
      autoRevive: false,
      previousMapId: undefined,
      battleLog: [],
      sessionStats: { startTime: Date.now(), kills: 0, bossKills: 0, expGained: 0, goldGained: 0, itemsDropped: 0 }
    };
  }
  
  private static saveCombatState(state: CombatState): void {
    SaveSystem.saveCombatState(state);
  }
}
```

### 5.1 阵亡/复活与安全区机制

- 阵亡后弹出全屏死亡面板，提供 `立即复活`、`回城（盟重土城）`、`自动复活` 三个操作。
- `立即复活`：消耗金币，恢复 50% HP/MP，原地图继续挂机。
- `回城`：免费传送到 `盟重土城`，恢复 100% HP/MP，并脱离战斗。
- `自动复活`：倒计时结束后自动按默认逻辑复活，适合持续挂机。
- 连续死亡达到阈值后，系统优先引导玩家回安全区，避免高难地图无限送死。
- 每次复活叠加 1 层虚弱，持续 30 秒；每层降低 10% 攻击/魔攻/防御/生命/魔法上限。

## 7. 离线战斗系统

```typescript
class OfflineCombatSystem {
  static readonly MAX_OFFLINE_HOURS = 8;
  static readonly OFFLINE_EFFICIENCY = 0.50; // 离线效率50%
  static readonly MAX_OFFLINE_DROP_ITEMS = 60; // 离线最大掉落件数（原为 20）
  
  /**
   * 计算并结算离线收益
   * 玩家登录时调用
   */
  static settleOfflineRewards(playerId: string): OfflineRewardResult {
    const player = PlayerSystem.getPlayerData(playerId);
    const now = Date.now();
    const lastTime = player.lastOfflineRewardAt;
    
    // 离线时长（分钟），上限8小时
    const offlineMinutes = Math.min(
      (now - lastTime) / 60000,
      this.MAX_OFFLINE_HOURS * 60
    );
    
    if (offlineMinutes < 1) return { exp: 0, gold: 0, items: [], minutes: 0 };
    
    // 获取玩家当前挂机地图
    const combatState = SaveSystem.loadCombatState(playerId);
    const mapId = combatState?.mapId ?? 'map_biqi';
    const mapConfig = MAP_CONFIG[mapId];
    
    const playerStats = PlayerSystem.getFinalStats(playerId);
    
    // 在线时经验/分钟
    const onlineExpPerMin = mapConfig.expPerMinute * (1 + playerStats.expBonus);
    const onlineGoldPerMin = mapConfig.goldPerMinute * (1 + playerStats.goldBonus);
    
    // 离线收益
    const exp  = Math.floor(onlineExpPerMin  * offlineMinutes * this.OFFLINE_EFFICIENCY);
    const gold = Math.floor(onlineGoldPerMin * offlineMinutes * this.OFFLINE_EFFICIENCY);
    
    // 离线掉落（概率较低）
    const items: EquipmentInstance[] = [];
    const dropAttempts = Math.floor(offlineMinutes / 5); // 每5分钟一次掉落机会
    for (let i = 0; i < dropAttempts; i++) {
      const drop = DropSystem.rollDrop(mapId, 'monster_normal', playerStats.magicFind * 0.3);
      if (drop.items.length > 0) items.push(...drop.items);
    }
    if (items.length > this.MAX_OFFLINE_DROP_ITEMS) {
      items.splice(this.MAX_OFFLINE_DROP_ITEMS); // 上限 60 件，原为 20
    }
    
    // 发放奖励
    PlayerSystem.addExp(playerId, exp);
    EconomySystem.addGold(playerId, gold);
    items.forEach(item => EquipmentSystem.addToInventory(playerId, item));
    
    // 更新时间戳
    SaveSystem.updateOfflineRewardTime(playerId, now);
    
    return { exp, gold, items, minutes: Math.floor(offlineMinutes) };
  }
}

interface OfflineRewardResult {
  exp: number;
  gold: number;
  items: EquipmentInstance[];
  minutes: number;
}
```

- 离线经验与金币按在线同地图收益的 **50%** 效率折算（旧版为 30%）。
- 离线结算的装备掉落件数上限为 **60** 件（旧版为 20）。

## 8. AI开发提示

### 8.1 实现优先级

1. **P0**：挂机循环（TICK逻辑）、伤害计算、经验/金币结算
2. **P1**：BOSS战、离线收益结算、战斗日志
3. **P2**：怪物技能、多目标群攻

### 8.2 关键约束

- TICK间隔建议用 `setInterval(1000)`，UI 每帧读取战斗状态渲染
- 挂机切换地图时先 `stopCombat`，再 `startCombat`
- 玩家死亡不立即原地复活，而是进入阵亡态；安全区地图不可触发战斗
- 自动补给与战斗循环解耦，避免在 `CombatSystem` 内部形成药剂循环依赖
- 离线奖励在每次登录时结算一次，结算后更新时间戳防止重复
- 战斗日志只保留最近 20 条，减少内存占用
