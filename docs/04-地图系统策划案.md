# 地图系统策划案

## 1. 系统概述

地图系统管理玩家的地图解锁进度、地图切换、BOSS刷新状态与安全区逻辑。当前版本除常规战斗地图外，额外包含 1 张 `盟重土城` 安全区地图，用于阵亡回城、脱战修整与安全停留。

### 1.1 当前版本迭代同步

- 新增 `盟重土城` 作为安全区地图，默认开放。
- 安全区地图 `isSafeZone = true`，不生成怪物、不产出经验金币、不触发战斗循环。
- 玩家阵亡后可选择“回城”并传送至安全区满状态复活。
- 地图顺序中安全区排在最前，便于 UI 地图页快速进入。

### 3.1 安全区地图规则

- `盟重土城` 不参与挂机收益计算，`expPerMinute = 0`、`goldPerMinute = 0`。
- 安全区不生成普通怪与BOSS，进入后 `CombatSystem.startCombat` 应直接停止战斗并显示安全提示。
- 阵亡回城会直接切换当前地图为 `map_tucheng`，并保留上一次危险地图 `previousMapId` 供后续返回参考。
- `MAP_ORDER` 中应将 `map_tucheng` 放在首位，确保地图页优先展示。

## 4. 核心接口

```typescript
class MapSystem {
  /** 获取玩家地图进度 */
  static getProgress(playerId: string): MapProgress;
  
  /** 检查地图是否已解锁 */
  static isUnlocked(playerId: string, mapId: string): boolean;
  
  /** 切换地图（含合法性校验） */
  static switchMap(playerId: string, mapId: string): SwitchMapResult;
  
  /** 解锁地图（达成条件时调用） */
  static unlockMap(playerId: string, mapId: string): void;
  
  /** 检查并触发地图解锁（每次升级/击杀BOSS后调用） */
  static checkAndUnlock(playerId: string): string[];  // 返回新解锁的地图ID列表
  
  /** 记录BOSS击杀 */
  static recordBossKill(playerId: string, bossId: string): void;
  
  /** 获取BOSS刷新剩余时间（秒） */
  static getBossRespawnTime(playerId: string, bossId: string): number;
  
  /** 获取某地图当前可用BOSS列表 */
  static getAvailableBosses(playerId: string, mapId: string): BossAvailability[];
}

interface SwitchMapResult {
  success: boolean;
  message?: string;   // 失败原因，如"等级不足，需要达到X级"
  mapId?: string;
}

interface BossAvailability {
  bossId: string;
  bossName: string;
  isAvailable: boolean;
  respawnSecondsLeft: number;   // 0 = 可挑战
  isFirstKill: boolean;         // 首杀奖励是否未领取
}
```

## 5. 解锁逻辑

```typescript
class MapUnlockChecker {
  static checkAll(playerId: string): string[] {
    const progress = MapSystem.getProgress(playerId);
    const player = PlayerSystem.getPlayerData(playerId);
    const newlyUnlocked: string[] = [];
    
    for (const [mapId, config] of Object.entries(MAP_CONFIG)) {
      if (progress.unlockedMaps.includes(mapId)) continue;
      if (!config.unlockCondition) continue;
      
      const cond = config.unlockCondition;
      let met = false;
      
      if (cond.type === 'level') {
        met = player.level >= (cond.value as number);
      } else if (cond.type === 'boss_kill') {
        met = progress.bossKillRecords.some(r => r.bossId === cond.value && r.totalKills > 0);
      }
      
      if (met) {
        MapSystem.unlockMap(playerId, mapId);
        newlyUnlocked.push(mapId);
      }
    }
    
    return newlyUnlocked;
  }
}
```

### 5.1 阵亡回城联动规则

- 玩家在非安全区阵亡后，可从死亡弹窗直接执行“回城”。
- 回城成功后：
  1. 当前地图改为 `map_tucheng`
  2. 玩家恢复满 HP/MP
  3. 立即停止当前战斗
  4. 保留阵亡前地图作为返回参考
- 若玩家连续多次阵亡，系统可优先建议停留安全区，避免战力不足时反复进入高压地图。

## 6. AI开发提示

### 6.1 实现优先级

1. **P0**：地图切换、解锁检查、BOSS刷新计时
2. **P1**：地图列表 UI 展示、推荐等级高亮
3. **P2**：地图内小地图、传送点

### 6.2 关键约束

- 切换地图时，先 `CombatSystem.stopCombat`，再 `MapSystem.switchMap`，最后 `CombatSystem.startCombat`
- BOSS 刷新时间存储为绝对时间戳，客户端倒计时 = `nextRespawnTime - Date.now()`
- 首杀奖励只发放一次，`isFirstKillDone` 标记为 true 后不再发放
- `isSafeZone=true` 的地图不得生成怪物、不得结算离线战斗、不得显示BOSS入口
